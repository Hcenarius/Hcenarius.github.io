定义

  • 正则表达式，又叫规则表达式,可以这么理解:有一定规律的表达式

作用

  • 匹配字符串的

由基本的一些符号组合成的,这些符号叫:基本的元字符

元字符

1.  . 表示的是除了\n以外的任意一个单个的字符
2. []  表示的是一个范围 ,只能是一个
3. [0-9] 表示的是所有的数字中 0到9之间的任意的一个数字(可以是0，也可以是9)
4. [a-z] 表示的是字母a到字母z之间的所有的小写字母中一个
5. [A-Z] 表示的是所有的大写的字母中的一个
6. [a-zA-Z] 所有的字母中的一个
7. [0-9a-zA-Z] 所有的数字和字母中的一个

[.] 干掉了这个.原有的意义了   就是一个普通的点


 |  或者
[0-9]|[a-z]  要么是一个数字,要么是一个小写的字母
[a]|[0-9]   要么是a要么是0到9之间的一个数字

() 提升优先级,分组
([a-z]|[0-9])[A-Z]
(([a]|[1-6])([A-Z][5])) 有三组 从最左边开始计算
()(())((()))

限定符

  1. +这个符号前面的表达式最少出现1次,最多到无限次 [0-9][a-z]+ “4f”
  2. 这个符号前面的表达式最少出现0次,最多到无限次 [a-zA-Z][2-9] “f9s”

  3. ? 这个符号前面的表达式出现的次数是0次到1次

    [0-9]?  "45"

  4. {1,} 此时的大括号表示的是前面的表达式出现了1次到多次

  5. {1,9} 表示前面的表达式出现了最少是1次,最多是9次 [2-8]{2,9}

  6. {0,1} 表示的是前面的表达式出现了最少0次，最多1次

     \d   表示的是一个数字,相当于[0-9]
 \D  表示的非数字

 \s   空白符号                       "   "
 \S   非空白符号
 \w    非特殊符号 包括_
 \W    特殊符号

^   以什么开始   ^[0-9]    还有可以表示 取非   [^0-9]  非数字

$   以什么结尾    [a-z]$   "2dfssd"
^[0-9][a-z]$  严格模式的匹配    "44444ufffff"

    例如

  • ip地址
  • 192.168.173.234
  • (\d{3}[.]){3}\d{3}

正则表达式在页面中的使用

创建对象才能使用正则表达式

两种方式创建正则表达式

    1. 通过调用构造函数创建对象
      • var reg=new RegExp(/正则表达式/);
    1. 字面量的方式创建正则表达式对象
      • var reg=/正则表达式/;

例如

创建正则表达式对象

var reg=new RegExp(/\d/);
//调用方法,返回值是bool类型
var flag= reg.test("10086");//判断这个正则表达式和字符串中的内容是否能够匹配
console.log(flag);//true

字面量的方式创建正则表达式对象

var reg=/[a-z]{1,3}/;
var flag=reg.test("我好开森哦,jack,do you 开心吗");
console.log(flag);

总结

  2. 之后的add每次进行调用都会在使用完毕后进行销毁，每次调用会调取fun当中的n，进行修改n ++ ，最后进行n的赋值，最后销毁
  3. 这条完整的链一直存在，只是add在不断的销毁或者重生，链条保证了n的存在，n的存在是由于fun存在，所以说fun的大环境才是闭包
  4. 真正的耗性能也是由于fun一直在内存空间当中没有被释放造成的，而add函数则是跟着result的调用而不断的销毁、重生。

Canvas 基础篇

定义

  • canvas 是 HTML5 提供的一个用于展示绘图效果的标签. canvas 原意画布, 帆布. 在 HTML 页面中用于展示绘图效果. 最早 canvas 是苹果提出的一个方案, 今天已经在大多数浏览器中实现.

    基本用法

    基本语法

  1. 使用 canvas 标签, 即可在页面中开辟一格区域. 可以设置其 width 和 height 设置该区域的尺寸.
  2. 默认 canvas 的宽高为 300 和 150.
  3. 不要使用 CSS 的方式设置宽高, 应该使用 HTML 属性.
  4. 如果浏览器不支持 canvas 标签, 那么就会将其解释为 div 标签. 因此常常在 canvas 中嵌入文本, 以提示用户浏览器的能力.
  5. canvas 的兼容性非常强, 只要支持该标签的, 基本功能都一样, 因此不用考虑兼容性问题.
  6. canvas 本身不能绘图. 是使用 JavaScript 来完成绘图. canvas 对象提供了各种绘图用的 api.

    使用领域

  7. 游戏
  8. 可视化数据(重点)
  9. banner 广告
  10. 多媒体
  11. 未来
    • 模拟仿真
    • 远程操作
    • 图形编辑

      基本绘图

      绘图的步骤

  12. 获得 canvas 对象.
  13. 调用 getContext 方法, 提供字符串参数 ‘2d’.
  14. 该方法返回 CanvasRenderingContext2D 类型的对象. 该对象提供基本的绘图命令.
  15. 使用 CanvasRenderingContext2D 对象提供的方法进行绘图.
  16. 基本绘图命令
    • 设置开始绘图的位置: context.moveTo( x, y ).
    • 设置直线到的位置: context.lineTo( x, y ).
    • 描边绘制: context.stroke().
    • 填充绘制: context.fill().
    • 闭合路径: context.closePath().

拿到绘制 3D 图形的绘图工具

canvas.getContext( 'webgl' )

结论

  1. 如果希望图片胖一点, 可以将 x 的值缩小( 是计算的值 ), x 缩小多少倍, 表示计算的 x 取值要扩大多少倍
  2. 如果希望图片往左移动, 即 x 负方向, 就将 x 的就算值 加上一个数字. 如果希望往右移动, 就让 x 减去一个数字.
    一旦出现 计算中 的 x 减去一个数字以后, 我们 x 的取值范围需要变化.
  3. 如果希望 图像往上移动, 即负方向移动, 就在 计算 y 值的时候 减去一个数字. 
    如果往下移动, 即往正方向移动, 就在计算 y 值的时候 加上一个数字.
  4. 如果希望 y 的图像上下拉伸, 变大就把计算 乘以一个数字, 如果变小就 除以一个数字. ( 与第一条类似 )

修改绘制颜色

  • 可以使用 context.strokeStyle = CSS 颜色语法来修改绘制的颜色. 需要注意的事情是 我们一旦修改需要 beginPath 一下
  • 如果修改的绘制的样式( 颜色, 线宽, 线帽, 拐点, 虚线等特征 ), 在 beginPath 之后依旧会保留下来

填充 fill

  • 在 绘制的时候, 我们可以连线一个 封闭的区域. 使用 fill 方法将这个封闭的区域填充起来， 如果需要修改填充的颜色, 可以使用 fillStyle.
  • 使用 fill, 如果绘制的不是闭合曲线, 那么 fill 会根据，我们绘制的起始点 和 最后的笔尖所在的位置 连接填充.

在填充的时候, 我们的绘制路径有交叉

非零环绕原则:

在绘制填充的时候, 首先有一点, 只会填充闭合的区域. 如果没有闭合会
默认将开始与结束相连构成闭合. 一旦闭合以后, 可能会将闭合区域分割中很多部分.
如果将闭合区域变成了多个部分, 那么判断这个部分要不要填充就可以使用 非零环绕原则:

  1. 在这个区域内任意的选一点( 越简单, 越孤立越好 )
  2. 以选择的这个点为圆心, 绘制一条完全穿过图形的射线( 越简单越好 )
  3. 以这个点为圆心, 看穿过射线的线的方向( 以圆周方向 )
  4. 将顺时针记为 +1, 将逆时针记为 -1, 最后看所有穿过的线的对应数字求和.
  5. 如果求和的结果为 0 即不填充, 如果求和的结果 非零 0, 则填充.

绘制虚线

设置虚线的时候就是在设置实线部分与空白部分的大小. 利用数组来描述其中的关系

例如: 
数组 [ 10, 10 ] 表示实线部分长度 10 像素, 空白部分长度 10 像素
数组 [ 10, 5 ] 表示实线部分 10 像素, 空白部分 5 像素
  • 如果给定的数组是奇数项, 那么实线空白部分轮询
    数组 [ 10, 5, 3 ] 则表示先实线部分 10 个像素, 空白部分 5 个像素,
    然后实线部分 3 个像素, 空白部分 10 个像素,
    再实现部分 5 个像素, 空白部分 3 个像素,
    …
    也就是说相当于 [ 10, 5, 3, 10, 5, 3 ]
  • 我们在绘制虚线的时候, 技巧就在于如何得到这个数组.利用 context.setLineDash( 数组 ), 如果要将虚线改为 实线 则使用空数组即可
