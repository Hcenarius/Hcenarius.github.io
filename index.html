正则表达式

Share

旁白

  • 可能是用的不多吧，熟练度不够，以至于到用的时候经常忘记，就在这里记录一下吧

定义

  • 正则表达式，又叫规则表达式,可以这么理解:有一定规律的表达式

作用

  • 匹配字符串的

由基本的一些符号组合成的,这些符号叫:基本的元字符

元字符

1.  . 表示的是除了\n以外的任意一个单个的字符
2. []  表示的是一个范围 ,只能是一个
3. [0-9] 表示的是所有的数字中 0到9之间的任意的一个数字(可以是0，也可以是9)
4. [a-z] 表示的是字母a到字母z之间的所有的小写字母中一个
5. [A-Z] 表示的是所有的大写的字母中的一个
6. [a-zA-Z] 所有的字母中的一个
7. [0-9a-zA-Z] 所有的数字和字母中的一个

[.] 干掉了这个.原有的意义了   就是一个普通的点


 |  或者
[0-9]|[a-z]  要么是一个数字,要么是一个小写的字母
[a]|[0-9]   要么是a要么是0到9之间的一个数字

() 提升优先级,分组
([a-z]|[0-9])[A-Z]
(([a]|[1-6])([A-Z][5])) 有三组 从最左边开始计算
()(())((()))

限定符

  1. +这个符号前面的表达式最少出现1次,最多到无限次 [0-9][a-z]+ “4f”
  2. 这个符号前面的表达式最少出现0次,最多到无限次 [a-zA-Z][2-9] “f9s”

  3. ? 这个符号前面的表达式出现的次数是0次到1次

    [0-9]?  "45"

  4. {1,} 此时的大括号表示的是前面的表达式出现了1次到多次

  5. {1,9} 表示前面的表达式出现了最少是1次,最多是9次 [2-8]{2,9}

  6. {0,1} 表示的是前面的表达式出现了最少0次，最多1次

     \d   表示的是一个数字,相当于[0-9]
 \D  表示的非数字

 \s   空白符号                       "   "
 \S   非空白符号
 \w    非特殊符号 包括_
 \W    特殊符号

^   以什么开始   ^[0-9]    还有可以表示 取非   [^0-9]  非数字

$   以什么结尾    [a-z]$   "2dfssd"
^[0-9][a-z]$  严格模式的匹配    "44444ufffff"

    例如

  • ip地址
  • 192.168.173.234
  • (\d{3}[.]){3}\d{3}

正则表达式在页面中的使用

创建对象才能使用正则表达式

两种方式创建正则表达式

    1. 通过调用构造函数创建对象
      • var reg=new RegExp(/正则表达式/);
    1. 字面量的方式创建正则表达式对象
      • var reg=/正则表达式/;

例如

创建正则表达式对象

var reg=new RegExp(/\d/);
//调用方法,返回值是bool类型
var flag= reg.test("10086");//判断这个正则表达式和字符串中的内容是否能够匹配
console.log(flag);//true

字面量的方式创建正则表达式对象

var reg=/[a-z]{1,3}/;
var flag=reg.test("我好开森哦,jack,do you 开心吗");
console.log(flag);

一个闭包的从生到死

“变量找寻链，这条链存在，找寻链经过的所有环境就不会被销毁”

总结

  1. 也就是说，闭包是一个大环境，fun被调用一次，这个环境就被创建并且在独立的内存空间当中一直存在，这就保证了n一直存在
  2. 之后的add每次进行调用都会在使用完毕后进行销毁，每次调用会调取fun当中的n，进行修改n ++ ，最后进行n的赋值，最后销毁
  3. 这条完整的链一直存在，只是add在不断的销毁或者重生，链条保证了n的存在，n的存在是由于fun存在，所以说fun的大环境才是闭包
  4. 真正的耗性能也是由于fun一直在内存空间当中没有被释放造成的，而add函数则是跟着result的调用而不断的销毁、重生。

    • 解决：
      我是 git config –global user.name “hcenarius”
      我的邮箱 git config –global user.email “lsylsf19941021.gmail.com”
      (PS: 你别真的把 我是。。。 我的邮箱。。打上去了)

      git config –list
      然后生成秘钥 ssh-keygen -t rsa -C “lsylsf19941021.gmail.com” 应该是连续按三四个回车就OK 如果中间弹出个（y/n）输入个 y 接着回车

      接着去github点你的头像，然后点下拉中的 Settings 然后看左边，有个SSH and GPG keys 点进去，

  • 问题2：报错出现

    • 1、 error:failed to execute prompt script (exit code 1) (那么恭喜你，麻烦了)

    • 原因：我猜测是配置文件的问题，也就是上面我们配置_config.yml文件的时候，没有配置好。

    • 解决：检查你的配置文件
      deploy:
      type: git
      repository: 你仓库地址（HTTPS开头的）
      branch: master
      看看格式是否正确 冒号后面必须跟空格，这几句的对齐方式也必须是这样的！！下面的缩进是两个空格，请不用使用 tab ！
      检查完没问题，好的，再去执行一遍hexo deploy，还是报错，那么接着试

      把配置文件改成这样：
      deploy:
      type: git
      repo: 你仓库地址 （git开头的）
      branch: master
      name: 你github上的用户名
      email: 你登陆github的邮箱
      （PS：取仓库地址—-> 登陆上去后，在右上角点你的github头像，下拉菜单中点your profile，
      然后找你刚才建的以你应户名然后.github.io结尾的仓库，点进去，右边有个绿色按钮，
      点击后，下拉的面板中左上角出现Clone with HTTPS,那下面的文本框中就是你以HTTPS开头的仓库地址，
      要使用git开头的，请点下拉面板中的右上角 Use SSH 然后复制下面的地址）
